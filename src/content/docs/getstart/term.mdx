このページでは、あしたぼホームページの開発ドキュメントで使われる専門用語や略語などを解説します。
Web開発やプログラミングに馴染みのない方でも理解しやすいように、できるだけ平易な言葉で説明することを心がけています。

もし分からない用語があれば、まずこのページを確認してみてください。

## あ行

### Astro (アストロ)
ウェブサイトを速く表示するための技術の一つです。たくさんの部品（コンポーネントと言います）からウェブページを作るのを手伝ってくれます。

### API (エーピーアイ)
「Application Programming Interface」の略です。ソフトウェアやプログラム同士が情報をやり取りするためのルールや窓口のようなものです。例えば、天気予報サイトが天気情報を提供するAPIを用意していれば、他のアプリはそのAPIを通じて天気情報を取得できます。

### Auth.js (オースジェイエス) / NextAuth.js (ネクストオースジェイエス)
ウェブサイトやアプリケーションにログイン機能（認証機能）を簡単に追加するためのライブラリです。GoogleアカウントやGitHubアカウントを使ったログインなどを手軽に実装できます。「あしたぼホームページ」でも使われています。

### ORM (オーアールエム)
「Object-Relational Mapper（オブジェクトリレーショナルマッパー）」の略です。プログラムの中のデータ（オブジェクト）と、データベースの中の表形式のデータをうまく対応付けてくれて、データベースの操作を簡単にしてくれる道具のことです。PrismaもORMの一種です。

### アクセシビリティ (Accessibility / a11y)
ウェブサイトやアプリが、年齢や身体的な条件に関わらず、誰にとっても使いやすいように設計・開発されている度合いのことです。「a11y」は "Accessibility" の最初と最後の文字 "A" と "y" の間に11文字あることから来ています。

### インタラクティブ (Interactive)
「双方向の」「対話的な」といった意味です。ウェブページで言えば、ユーザーがボタンをクリックしたり、何かを入力したりすると、ページがそれに反応して見た目を変えたり、新しい情報を表示したりすることです。ゲームのようにユーザーの操作に応じて動的に変化するページはインタラクティブ性が高いと言えます。

### ISR (アイエスアール)
「Incremental Static Regeneration（インクリメンタル静的再生成）」の略です。Next.jsなどで使われるページの表示方法の一つで、基本的には事前に作っておいたページ（SSG）を表示しつつ、必要に応じて定期的に裏側でページを新しく作り直して更新する仕組みです。速さと新しさを両立できます。

## か行

### Git (ギット)
ファイルの変更履歴を記録・管理するためのシステムです。誰がいつどこを変更したかが分かるので、複数人での開発や、過去の状態に戻したい場合に便利です。

### GitHub (ギットハブ)
Gitで管理しているプロジェクトをインターネット上で共有・共同作業するためのサービスです。

### クライアント (Client)
Webの世界では、主にユーザーが直接操作する側のコンピュータやソフトウェア（例: Webブラウザ、スマートフォンアプリ）のことを指します。サーバーに情報やサービスを要求（リクエスト）する側です。「フロントエンド」とほぼ同じ意味で使われることもあります。

### コミット (Commit)
Gitで、ファイルの変更内容を記録する操作のことです。「この変更を保存します」という意思表示のようなもので、変更内容と説明（コミットメッセージ）をセットで記録します。

### コンポーネント
ウェブページを構成する部品のことです。例えば、ボタンや入力フォーム、ヘッダーやフッターなどがコンポーネントにあたります。同じような部品を何度も作らなくて済むように、再利用可能な形でまとめられています。

### コンフリクト (Conflict)
Gitで、複数の人が同じファイルの同じ部分を別々に変更してしまった場合に発生する「衝突」のことです。どちらの変更を優先するのか、あるいは両方を取り込むのかを人間が判断して解決する必要があります。

## さ行

### Starlight (スターライト)
Astroを使ってドキュメントサイト（説明書のようなウェブサイト）を簡単に作るための道具（テーマやコンポーネントの集まり）です。この開発ドキュメントもStarlightを使っています。

### サーバー (Server)
Webの世界では、クライアント（Webブラウザなど）からの要求に応じて、情報やサービスを提供する側のコンピュータやソフトウェアのことを指します。ウェブページや画像ファイルを保管しておき、リクエストがあればそれを送り返したり、データベースと連携してデータを処理したりします。「バックエンド」とほぼ同じ意味で使われることもあります。

### CSR (シーエスアール)
「Client-Side Rendering（クライアントサイドレンダリング）」の略です。ウェブページの表示内容を、主にユーザーのWebブラウザ側（クライアント側）でJavaScriptを使って組み立てる方法です。

### CLI (シーエルアイ)
「Command Line Interface（コマンドラインインターフェース）」の略です。キーボードから文字で命令（コマンド）を打ち込んでコンピュータを操作する方法です。黒い画面に文字だけが表示される「ターミナル」や「コマンドプロンプト」などがこれにあたります。マウスでアイコンをクリックするGUI（グラフィカルユーザーインターフェース）の対義語です。

### SSG (エスエスジー)
「Static Site Generation（静的サイトジェネレーション）」の略です。ウェブサイトを公開する前に、あらかじめ全てのページをHTMLファイルとして作成しておく方法です。表示が非常に速いのが特徴です。

### SSR (エスエスアール)
「Server-Side Rendering（サーバーサイドレンダリング）」の略です。ユーザーがウェブページにアクセスするたびに、サーバー側でそのページの内容を組み立てて（レンダリングして）からユーザーのブラウザに送る方法です。常に最新の情報を表示できるのが特徴です。

## た行

### TypeScript (タイプスクリプト)
JavaScriptというプログラミング言語をより安全で大規模な開発にも使いやすくしたものです。プログラムの間違いを早期に発見しやすくする機能があります。

### Tailwind CSS (テイルウィンド シーエスエス)
ウェブサイトのデザインを整えるための道具（CSSフレームワーク）の一つです。「ユーティリティクラス」という短い命令をHTMLに直接書き込むことで、見た目を細かく調整できます。

### daisyUI (デイジーユーアイ)
Tailwind CSSをさらに使いやすくするための部品集です。ボタンやカードなど、よく使われるデザイン部品があらかじめ用意されています。

### データベース (Database)
たくさんの情報を整理して保存し、必要な時に取り出したり更新したりできるようにしておくための仕組みやシステムのことです。電話帳や図書館の蔵書リストのようなものです。

### Docker (ドッカー)
アプリケーションを「コンテナ」という隔離された環境で動かすための技術です。自分のPCに直接いろいろな開発ツールをインストールしなくても、コンテナに必要なものをまとめておくことで、どこでも同じように開発環境を再現できます。

## な行

### Next.js (ネクストジェイエス)
Reactを使ってウェブアプリケーションを作るための人気のフレームワーク（開発を効率化するための骨組みや道具の集まり）です。ウェブサイトの表示速度を上げたり、開発をしやすくしたりする機能がたくさん備わっています。あしたぼホームページはこのNext.jsで作られています。

### Node.js (ノードジェイエス)
通常はウェブブラウザの中で動くJavaScriptというプログラミング言語を、自分のコンピュータ上でも動かせるようにするものです。ウェブサーバーを動かしたり、開発に必要なツールを実行したりするのに使われます。

### 認証 (Authentication)
サービスを利用しようとしている人が、本当にその本人であるかを確認する手続きのことです。IDとパスワードの入力や、指紋認証などがこれにあたります。Auth.jsはこれを手助けするツールです。

## は行

### Prisma (プリズマ)
データベース（情報を整理して保存しておく場所）とプログラムを繋ぐための道具（ORM: Object-Relational Mapper）です。データベースの操作を簡単かつ安全に行えるようにします。

### PostgreSQL (ポストグレスキューエル)
高性能で信頼性の高いデータベース管理システムの一つです。あしたぼホームページでは、ユーザー情報や予約情報などを保存するために使われています。

### バックエンド (Backend)
ウェブサイトやアプリケーションの「裏側」の部分で、ユーザーの目には直接見えない処理を担当します。データベースとの連携、ユーザー認証、複雑な計算処理などがこれにあたります。サーバーサイドとも呼ばれます。

### バージョン管理システム (Version Control System)
ファイルの変更履歴を記録・管理し、過去の状態に戻したり、複数人での共同作業をスムーズに行えるようにしたりするためのシステムです。Gitが代表的です。

### プッシュ (Push)
Gitで、自分のコンピュータ（ローカルリポジトリ）に記録した変更内容（コミット）を、インターネット上の共有場所（リモートリポジトリ、例: GitHub）に送信して反映させる操作のことです。

### プル (Pull)
Gitで、インターネット上の共有場所（リモートリポジトリ）にある最新の変更内容を、自分のコンピュータ（ローカルリポジトリ）に取り込んで同期させる操作のことです。

### プルリクエスト (Pull Request / PR)
GitおよびGitHub（などのホスティングサービス）で、自分が行った変更（通常は特定のブランチでの変更）を、他の人にレビューしてもらい、問題がなければメインのブランチに取り込んでもらうためのお願い（リクエスト）のことです。

### フレームワーク (Framework)
アプリケーション開発を効率的に行うための「骨組み」や「枠組み」のことです。よく使われる機能や基本的な構造があらかじめ用意されており、開発者はそれに沿って必要な部分を作ることで、手間を減らし、品質の高いアプリケーションを短期間で作ることができます。Next.jsもフレームワークの一種です。

### フロントエンド (Frontend)
ウェブサイトやアプリケーションの「表側」の部分で、ユーザーが直接目にしたり操作したりする部分（ユーザーインターフェース）のことです。HTML、CSS、JavaScript（Reactなど）を使って作られます。クライアントサイドとも呼ばれます。

### ブランチ (Branch)
Gitで、開発作業を分岐させて、元のコードに影響を与えずに新しい機能の追加やバグ修正などを並行して行うための仕組みです。「木の枝」のように、メインの流れから分岐して作業を進め、完了したらまた合流（マージ）させることができます。

## ま行

### Markdown (マークダウン)
簡単な記号を使って文章の構造（見出し、箇条書きなど）を表現できる書き方です。HTMLというウェブページを作るための言語に変換されて表示されます。このドキュメントも主にMarkdownで書かれています。

### マージ (Merge)
Gitで、分岐したブランチで行った変更を、別のブランチ（通常はメインのブランチ）に統合（合流）させる操作のことです。

## ら行

### React (リアクト)
ウェブページの見た目（ユーザーインターフェース）を作るためのJavaScriptのライブラリ（便利な機能を集めたもの）です。コンポーネントという部品を組み合わせて効率的に開発できます。

### ライブラリ (Library)
プログラミングにおいて、よく使われる便利な機能や部品をまとめたものです。自分で一から全部作らなくても、ライブラリを読み込んで使うことで、効率的に開発を進めることができます。Reactもライブラリの一種です。

### ルーティング (Routing)
ウェブサイトやアプリケーションで、特定のURLにアクセスしたときに、どのページやどの機能を表示するかを対応付ける仕組みのことです。Next.jsのApp Routerなどがこれを担当します。

### レスポンシブデザイン (Responsive Design)
ウェブサイトのデザインを、パソコン、タブレット、スマートフォンなど、さまざまな画面サイズに合わせて見やすく、操作しやすいように自動的に調整する手法のことです。

### レンダリング (Rendering)
プログラムが書いたコード（HTML、CSS、JavaScriptなど）を元に、実際に画面に表示される内容を組み立てて描画することです。サーバーサイドレンダリング（SSR）やクライアントサイドレンダリング（CSR）などの方法があります。

### リポジトリ (Repository)
Gitで管理されているプロジェクトのファイルや変更履歴などをまとめて保管しておく場所のことです。自分のコンピュータ上にある「ローカルリポジトリ」と、GitHubなどのインターネット上にある「リモートリポジトリ」があります。

## わ行

### WSL (ダブリューエスエル)
「Windows Subsystem for Linux」の略です。Windowsの中でLinuxの環境を動かすための仕組みです。これにより、WindowsユーザーでもLinux用の開発ツールやコマンドを簡単に利用できます。
